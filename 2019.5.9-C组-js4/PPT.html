一、背景介绍

在javaScript中，this是动态绑定的，它可以是全局对象、当前对象或者任意对象，这完全取决于函数的调用方式。在绝大多数情况下，函数的调用方式决定了this的值,this指向调用函数的对象。this不能在执行期间被赋值，并且在每次函数被调用时this的值也可能会不同。

二、知识剖析

谁最终调用函数，this就指向谁

1、this指向的，永远只可能是对象！

2、this指向谁，永远不取决于this写在哪！而是取决于函数在哪调用。

3、this指向的对象，我们称之为函数的上下文context，也叫函数的调用者。

1,普通函数调用

全局作用域或者普通函数中 this 指向全局对象 window。

严格模式 ‘use strict’下的this

在严格模式下，在全局环境中执行函数调用的时候 this 并不会指向 window 而是会指向 undefined。

2,对象函数调用

this指向这个对象

3, 函数作为数组的一个元素，通过数组下标调用的

this指向这个数组

4,在构造函数或者构造函数原型对象中 this

this 指向构造函数的实例

作为构造函数被调用的时候，this 代表它即将 new 出来的对象

注：构造函数不使用new调用，则和普通函数一样。一般地，构造函数首字母大写 如果不加 new,表示即作为普通函数调用，指向 window

5,箭头函数中的 this

this指向外层作用域的 this

6,call和apply

简单来说就是会改变传入函数的this。主要作用有两个

1.改变 this 的指向

2.方法借用

唯一的区别就是在参数上面。

三、常见问题

setTimeout、setInterval中的this是指向哪里？

setTimeout、setInterval中的this再实际调用的时候this指向全局Window的对象

四、解决方案

《javascript 高级程序设计》中写到：“超时调用的代码都是在全局执行域中执行的”。

setTimeout/setInterval 执行的时候，this 默认指向 window 对象，除非手动改变 this 的指向。
